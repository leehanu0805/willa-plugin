# Phase 4.75 · Self-Heal (자동 · 무제한)

Investigate 게이트 통과 직후 **자동 진입**. 사용자 승인 없이 시작 · 무제한 반복 · 스마트 stop. 목적: **Plan 의도 ↔ 실제 배치/디자인 일치** + **조작성 자동 개선**.

## 왜 필요한가

- Scaffold 는 Plan 을 따라 코드를 작성하지만 놓치는 디테일이 있다
- Investigate 게이트는 사용자 확인이 우선 · 깊은 자가 검증은 부족
- Serve 로 사용자에게 보여주기 전에 **willa 자율 정리** 로 품질 safe-guard

**5 Serve 로 진입하기 전 이 phase 가 반드시 완료돼야 함.**

## 매 반복 (Iter N)

### 1. Plan 정렬 체크 (Plan Alignment)

`.willa/plan-latest.md` 읽고 각 항목이 실제 코드에 반영됐는지 검증:

| Plan 항목 | 체크 대상 | 불일치 예시 |
|----------|----------|------------|
| 디자인 토큰 (팔레트/타이포/spacing) | `src/index.css` 변수값 | plan #0A0908 vs 실제 #000000 |
| 레이아웃 치수 | TopBar height · Rail width 등 | plan 44px vs 실제 52px |
| 상태 다양성 | hover/active/disabled 구현 | disabled 누락 |
| 모션 토큰 | transition duration/easing | 명시만 있고 적용 안 됨 |
| 컴포넌트 배치 | Inspector/Dock 위치 · 기본 상태 | Inspector default 펼침인데 plan 은 접힘 |
| Archetype 기록 (Keep/Fix/Add) | 실제 변경 반영 | "Keep: Inspector 접힘" 인데 안 지켜짐 |

불일치 발견 → 즉시 수정 (Plan 쪽이 아닌 코드 쪽을 고침).

### 2. 조작성 휴리스틱 체크

`references/nng-heuristics.md` + `references/laws-of-ux.md` 기반 빠른 감사:

- **Fitts's Law**: 자주 쓰는 도구가 화면 가장자리/중앙 (큰 타겟)?
- **Hick's Law**: 최상위 진입점 ≤ 5-7 · TopBar/Rail/Inspector 탭 개수 초과 여부
- **3-click rule**: 임의 기능 도달이 3클릭 이내인가
- **NNG #1 Visibility**: 세그 진행률 · 선택 객체 · 현재 Phase 상시 가시?
- **NNG #6 Recognition > Recall**: 아이콘 단독 금지 · 라벨 병기
- **NNG #7 Flexibility**: 단축키 + 시각 진입점 병행

발견 → 즉시 수정.

### 3. TypeScript + 렌더 체크

각 수정 후 `npx tsc --noEmit` 통과 확인. 실패하면 해당 수정 롤백.
