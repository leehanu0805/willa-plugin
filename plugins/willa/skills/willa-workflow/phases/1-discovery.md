# Phase 1 · Discovery

willa의 목표: **지금 있는 Will3D 기능·플로우를 사용자가 더 쉽게 쓰도록 UI/UX를 수정**.

**Will3D = 치과 CBCT 3D 뷰어/플래너.** 이 도메인 제약은 절대 바뀌지 않음:
- 의사가 CT 볼륨을 3축 슬라이스로 스캔
- 임플란트 배치 · 각도/깊이 조정
- 측정(거리 · 각도 · HU)
- WW/WL · Clipping · 세그먼트 토글
- 3D 볼륨 렌더 회전
- 보고서/STL 내보내기

→ **뷰포트가 주인공인 인터랙티브 도구**. 저널 리더 · 대시보드 · 소셜 피드 같은 형태는 **부적합**.

## 읽기 대상

Will3D C++ 소스를 읽어 **현재 UX 문제점** 을 식별:

- `CTCommon\W3Enum.h` (TabType)
- `TabMgr\W3Tabmgr.cpp` (탭 전환 로직)
- `UIFrame\*.cpp` (타이틀바/툴바/탭 위젯 구조)
- `UIComponent\view_gl_*.h` (뷰 컴포넌트)
- `UITools\*_task_tool.h` (탭별 도구)

## 파악할 것 — 구조가 아니라 "문제"

단순히 "탭이 몇 개 있다" 가 아니라 **UX 이슈를 찾는다**:

| 질문 | 예시 답변 |
|------|----------|
| 자주 쓰는 기능이 찾기 어려운가? | 9개 탭이 주석 처리돼 숨겨짐 · Lightbox가 어디 있는지 모름 |
| 워크플로우 전환에 클릭 많은가? | MPR → Implant 넘어가려면 탭 3번 클릭 |
| 공간 배치 비효율? | 우측 OTF 패널이 Implant 탭에서도 고정 · 뷰 공간 침범 |
| 컨텍스트/선택 피드백 약한가? | 선택한 임플란트 속성이 어디 표시되는지 불명확 |
| 중복 작업 있나? | WW/WL 조정을 탭마다 다시 찾음 |
| 이름/라벨 혼란? | IMPANT_GUIDE 오타 · 기능명이 기술적 |
| 키보드 단축키 부재? | 슬라이스 스크롤/도구 선택 전부 마우스 |
| 새 기능이 구조에 어색하게 박힘? | 세그먼테이션이 MPR 탭 내부에만 있음 |

## 출력 (사용자에게 짧게)

```
Will3D 파악 완료. CBCT 뷰어/플래너.

구조 요약:
- 활성 탭 5 · 숨겨진 탭 9 · 뷰 25 · 툴 14

발견된 UX 이슈 (Top 5):
1. {구체 이슈 · C++ 코드 근거}
2. ...
...

다음: 어느 이슈부터 해결할지 질문 드릴게요.
```

## 금지

- **도메인 혼동 금지** — Will3D 를 저널/대시보드/소셜 앱으로 재해석하지 말 것
- **Aesthetic 먼저 선택 금지** — aesthetic 은 Plan 단계에서 이슈 해결책에 종속되어 결정됨
- 사용자에게 질문 금지 (Clarify 단계에서 일괄)
- C++ 본체 수정 시도 금지
- 5분 이상 탐색 금지
