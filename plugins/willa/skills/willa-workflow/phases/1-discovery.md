# Phase 1 · Discovery

willa의 목표: **지금 있는 Will3D 기능·플로우를 사용자가 더 쉽게 쓰도록 UI/UX를 수정**.

**Will3D = 치과 CBCT 3D 뷰어/플래너.** 이 도메인 제약은 절대 바뀌지 않음:
- 의사가 CT 볼륨을 3축 슬라이스로 스캔
- 임플란트 배치 · 각도/깊이 조정
- 측정(거리 · 각도 · HU)
- WW/WL · Clipping · 세그먼트 토글
- 3D 볼륨 렌더 회전
- 보고서/STL 내보내기

→ **뷰포트가 주인공인 인터랙티브 도구**. 저널 리더 · 대시보드 · 소셜 피드 같은 형태는 **부적합**.

## 읽기 대상

Will3D C++ 소스를 읽어 **현재 UX 문제점** 을 식별:

- `CTCommon\W3Enum.h` (TabType)
- `TabMgr\W3Tabmgr.cpp` (탭 전환 로직)
- `UIFrame\*.cpp` (타이틀바/툴바/탭 위젯 구조)
- `UIComponent\view_gl_*.h` (뷰 컴포넌트)
- `UITools\*_task_tool.h` (탭별 도구)

## 파악할 것 — 현재 UI/UX 상태

**문제 탐지가 아닌 "이해"**. 어떤 게 좋고 어떤 게 불편한지 관찰 기록. 개선 여부 판단은 Plan 에서.

| 관찰 항목 | 예시 |
|-----------|-------|
| 탭/네비게이션 구성 | 5 활성 · 9 주석처리 · 평면 나열 |
| 자주 쓰는 도구 위치 | MPR/Implant 탭 각각 ToolVBar 별도 |
| 워크플로우 전환 경로 | MPR → Implant 3 클릭 |
| 공간 배치 | OTF 패널 우측 고정 · 탭 무관 · 뷰 80% |
| 컨텍스트/선택 피드백 | 임플란트 클릭 시 속성은 우측 OTF 영역 교체 |
| 중복 작업 패턴 | WW/WL · 측정 도구 탭마다 재구성 |
| 이름/라벨 | IMPANT_GUIDE 오타 · 기능명 영문 기술적 |
| 단축키 체계 | File/Edit/Help 메뉴만 · Ctrl+K 팔레트 없음 |
| 자동화/AI 기능 | AutoSegmentator 는 MPR 탭 내부 버튼 |
| 시각/톤 | Dark · QT 기본 · 의료SW 스탠다드 |

## 출력 (사용자에게 짧게)

```
Will3D 파악 완료. CBCT 뷰어/플래너.

구조 요약:
- 활성 탭 5 · 숨겨진 탭 9 · 뷰 25 · 툴 14

현재 UI/UX 관찰 (개수 제한 없음):
- 구체 관찰 1 · C++ 코드 근거
- 구체 관찰 2
- ...

다음: 방향성 4개 고정 질문 드릴게요.
```

관찰 개수는 사실에 따라 · 3개든 10개든 상관없음.

## 금지

- **도메인 혼동 금지** — Will3D 를 저널/대시보드/소셜 앱으로 재해석하지 말 것
- **Aesthetic 먼저 선택 금지** — aesthetic 은 Plan 단계에서 이슈 해결책에 종속되어 결정됨
- 사용자에게 질문 금지 (Clarify 단계에서 일괄)
- C++ 본체 수정 시도 금지
- 5분 이상 탐색 금지
