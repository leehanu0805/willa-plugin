# Phase 2 · Clarify

Discovery 결과 보고 후 **어떤 방향으로 개선할지** 질문. `AskUserQuestion` 3~4개.

## 기본 질문 세트

### Q1. 우선순위 대상 사용자
현재 Will3D는 범용 치과 SW. 누굴 최적화할지.

| 옵션 | 의미 |
|------|------|
| 임플란트 전문의 | PLAN 중심 최적화 |
| 일반 치과의 (진단 중심) | EXAMINE · DELIVER 중심 |
| 교정 전문의 | Clear Aligner · Face Sim · Ceph 우대 |
| 범용 | 빈도 균형 |

### Q2. 개선 범위
얼마나 크게 바꿀지.

| 옵션 | 의미 |
|------|------|
| L1 · 배치 미세조정 | 현 구조 유지 · 도구 위치만 조정 |
| L2 · 레이아웃 재배치 | 패널 크기·위치·그룹핑 재정렬 |
| L3 · 구조 재설계 | Phase/Navigation 패러다임 재정의 |
| L4 · 완전 재시작 | 처음부터 새 IA (공격적) |

### Q3. 정보 밀도
한 화면에 얼마나 많이 보여줄지.

| 옵션 | 의미 |
|------|------|
| 프로 고밀도 | 한 화면에 많은 컨트롤 (Photoshop 식) |
| 현재 수준 | 지금 Will3D 정도 |
| 간소화 | 숨김 많이, progressive disclosure |

### Q4. 3-모드 출력
| 옵션 | 의미 |
|------|------|
| 3개 전부 (MOCKUP/WIREFRAME/GUIDE) | 모두 생성 |
| MOCKUP만 | 디자인만 |
| MOCKUP + GUIDE | 디자인 + 핀 설명 |

## 규칙

- 한 번에 최대 4개만
- 답변 받으면 1줄 요약으로 확인: "정리: 임플란트 전문의 · L2 · 현재 밀도 · 3모드 전체"
- 내부 메모로 유지 → Plan에서 참조

## 결정 로그

```
{
  targetUser: "임플란트 전문의",
  scope: "L2",
  density: "현재 수준",
  modes: ["mockup", "wireframe", "guide"],
}
```
